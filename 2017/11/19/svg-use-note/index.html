3.3 使用ng-repeat遇到的删除元素但样式仍被保留问题

代码中使用了ng-repeat来将一组元素映射为dom，但是在交互中遇到如果删除了repeat中的元素，原来解析出来的dom如果有新设置的样式或类，会保留下来，所以很诡异的现象：明明从页面上删除了某个元素，但是它的样式却传递给了上一个元素，如果各个元素样式都不同，那么会出现样式错乱。

解决：样式与元素一一对应，而不是依赖元素位置。或者删除元素前先恢复该元素的样式到默认样式。
